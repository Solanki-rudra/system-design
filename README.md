# System Design Learning Repository

Welcome to the System Design learning repository! This project is organized into several directories to help you sequentially learn, revise, and practice system design concepts and problems.

## 📂 Repository Structure

Navigate through the folders below to find notes, templates, and full system architectures.

### 1. [`fundamentals/`](./fundamentals/)
Contains fundamental concepts you need to understand before diving into large-scale system design.
* **Topics:** What is a system, basics of distributed systems, CAP Theorem, Caching, Load Balancing, etc.

### 2. [`core-concepts/`](./core-concepts/)
Focuses on the core principles and evaluation metrics used in system design interviews and architecture planning.
* **Topics:** Requirements gathering (Functional vs. Non-functional), scoping, tradeoffs, etc.

### 3. [`system-designs/`](./system-designs/)
Detailed architectures and explanations of popular real-world systems.
* **Examples:** URL Shortener, Todo App, Video Streaming, Banking Systems, etc.

### 4. [`interview/`](./interview/)
Resources to help you prepare specifically for system design interviews.
* **Contents:** Approach templates for structuring answers, common interview questions, and time management strategies.

### 5. [`practice/`](./practice/)
A dedicated space for practicing system design problems using structured approaches.
* **Contents:** Standard templates to use when practicing and your own architectural designs.

### 6. [`cheatsheets/`](./cheatsheets/)
Quick reference guides to quickly brush up on system design topics.
* **Contents:** Quick revision notes, diagram guidelines, and quick summaries.

---

## 🚀 How to Use

1. **Build the Foundation**: Start with `fundamentals/` to get familiar with basic building blocks.
2. **Understand the Rules**: Read through `core-concepts/` to understand how to approach, scope, and evaluate problems.
3. **Analyze Examples**: Review the complete architectures in `system-designs/` to see how all the concepts come together in real-world scenarios.
4. **Practice**: Test your skills using the templates located in the `practice/` folder.
5. **Interview Prep**: Use the `cheatsheets/` and `interview/` sections right before your interviews.
