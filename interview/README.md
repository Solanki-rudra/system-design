# Interview Preparation

Welcome to the **Interview** section! This directory is tailored specifically for helping you succeed in high-pressure system design interviews.

## 📂 File Directory

- [`approach-template.md`](./approach-template.md) - A systematic framework and structure you should follow to ace your 45-minute interviews.
- [`common-questions.md`](./common-questions.md) - A curated list of frequently asked system design problems and their core components.

## 💡 Key Insights
- Knowing the architecture is only 50% of the interview; the other 50% is *communication* and *structuring your answer*.
- Leading the conversation using a structured template ensures you don't stall or end up in rabbit holes.
- Focus on trade-offs: There is no "perfect" system, only a system that solves specific requirements better than others.

## 🚀 How to Use
1. **Master the Framework:** Go through [`approach-template.md`](./approach-template.md) and memorize the phase sequence (Gathering Requirements -> High-Level Design -> Deep Dive -> Bottlenecks).
2. **Practice Patterns:** Review [`common-questions.md`](./common-questions.md) to identify recurring patterns like rate limiting, sharding, and caching strategies.
3. **Simulate:** Keep the template open on a second monitor while doing mock interviews to practice your flow and time management.
